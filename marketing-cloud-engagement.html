    Marketing Cloud Engagement

    O Salesforce Marketing Cloud Engagement é a plataforma de marketing digital omnicanal da Salesforce. Ela permite planejar, executar e otimizar campanhas de marketing em múltiplos canais – email, SMS, push, redes sociais, web e publicidade – a partir de uma visão unificada do cliente. Esta página apresenta um guia completo de estudo, organizado por níveis de dificuldade, para apoiar a sua jornada rumo às certificações e práticas avançadas.

    Nível Iniciante

    Explicação: Nesta etapa você aprenderá a base do Marketing Cloud: entender o modelo de dados de contatos, criar listas e Data Extensions, segmentar assinantes e construir emails personalizados e responsivos.

    Caso de uso: Onboarding de um novo assinante: assim que alguém se cadastra na newsletter, envie um email de boas‑vindas personalizado e adicione-o a um segmento adequado.

    Tópicos teóricos:

    • Modelo de dados: Subscriber Key vs Contact Key, Listas x Data Extensions e criação de Attribute Groups.
    • Email Studio e Content Builder: templates, blocos de conteúdo, strings de personalização e conteúdo dinâmico básico.
    • Segmentação e assinaturas: filtros, criação de listas, regras de subscription e práticas de entregabilidade (opt‑in, opt‑out, CAN-SPAM).
    • Contact Builder: criação e importação de Data Extensions, uso do Data Designer para mapear relacionamentos.

    Hands‑on / Labs:

    • Configurar o Contact Builder e criar uma Data Extension “Newsletter_Subscribers” com campos SubscriberKey, EmailAddress, FirstName e PreferredCategory.
    • Importar contatos via CSV para a Data Extension e criar filtros simples de segmentação.
    • Criar um email de boas‑vindas no Content Builder com personalização %%FirstName%% e links de preferências.
    • Programar um envio de teste e analisar as métricas de abertura e cliques no Email Studio.

    Testes de conhecimento & Exercícios:

    • Explique quando usar Subscriber Key e quando usar Contact Key.
    • Crie uma segmentação que encontre assinantes que não abriram os últimos 3 envios.
    • Demonstre como garantir conformidade de opt‑in e inserir um link de descadastramento nos emails.

    Nível Intermediário

    Explicação: Você passa a orquestrar comunicações contínuas e multicanal usando Automation Studio e Journey Builder. Aprende a processar dados em escala, definir entradas de jornada e ramificações condicionais.

    Caso de uso: Conduzir leads ao longo de uma jornada de boas‑vindas: importar leads diariamente, enviar email inicial, aguardar alguns dias, enviar SMS de lembrete e seguir caminhos diferentes conforme a interação.

    Tópicos teóricos:

    • Automation Studio: atividades de Import File, Filter, SQL Query e Email Send, além de tarefas de Data Extract e File Transfer.
    • Journey Builder: fontes de entrada (Data Extensions, API Events), atividades (send email, SMS, push, advertising, waits, decisions) e interação com Sales Cloud via Marketing Cloud Connect.
    • SQL e segmentação avançada: escrever queries em Data Extensions e usar Data Views para acompanhar engajamento.
    • Integração básica com Mobile Studio (SMS e Push) e Advertising Studio.

    Hands‑on / Labs:

    • Criar uma automação no Automation Studio que importa novos leads de um FTP, grava numa Data Extension e inicia uma jornada.
    • Construir uma jornada de boas‑vindas no Journey Builder com e‑mails, waits, decision splits e envio de SMS via MobileConnect.
    • Escrever uma Query Activity para separar contatos ativos e inativos com base em Data Views.
    • Utilizar o Marketing Cloud Connect para sincronizar leads da Sales Cloud e mapear um campo customizado.

    Testes de conhecimento & Exercícios:

    • Quando usar uma atividade SQL Query em vez de um Data Filter? Explique as diferenças e limites.
    • Desenhe um fluxograma de jornada com dois caminhos, um para quem clicou no email inicial e outro para quem não clicou.
    • Crie uma automação que exporte uma Data Extension para o FTP todas as sextas‑feiras.

    Nível Avançado

    Explicação: Aprofunde‑se em scripting, APIs e recursos de inteligência. Você irá personalizar conteúdo com AMPscript e SSJS, usar APIs REST/SOAP para integrações em tempo real e explorar recursos de IA como Einstein.

    Caso de uso: Disparar emails transacionais a partir de eventos de compra no e‑commerce com conteúdo de recomendação de produtos alimentado por IA e dados externos.

    Tópicos teóricos:

    • AMPscript e SSJS: funções de lookup, lógica condicional, criação de blocos dinâmicos e scripts em Server‑Side JavaScript para automações.
    • APIs do Marketing Cloud: REST e SOAP para envio de Triggered Sends, criação de contatos, consultas de Data Extensions e autenticação via OAuth.
    • Data Views e relatórios: uso avançado de consultas para analisar comportamento (opens, clicks, bounces) e construir dashboards.
    • Einstein Engagement Scoring e Send Time Optimization: configurar e interpretar pontuações preditivas e horários otimizados.
    • Integração com sistemas externos via API ou middleware (ex.: capturar eventos de e‑commerce e disparar um email transacional).

    Hands‑on / Labs:

    • Criar um email com bloco de recomendação de produtos usando AMPscript que consulta uma Data Extension de catálogo.
    • Escrever uma Query Activity que combine dados de Data Views para identificar assinantes com alto engajamento.
    • Configurar um Triggered Send via REST API e testar o envio com Postman.
    • Ativar o Einstein Send Time Optimization em uma jornada e analisar os resultados.

    Testes de conhecimento & Exercícios:

    • Diferencie Triggered Send, User‑Initiated Send e Journey Send.
    • Escreva um snippet de AMPscript que exibe um desconto de 10% para assinantes VIP.
    • Explique como autenticar e fazer uma chamada REST API para criar um contato no Marketing Cloud.

    Nível Especialista/Consultor

    Explicação: Neste nível, você age como estrategista. Entende como unir várias nuvens Salesforce, alavancar IA e personalização em escala, e desenhar arquiteturas de marketing omnicanal robustas para milhões de contatos.

    Caso de uso: Lançar um novo produto com uma campanha integrada que inclui email, SMS, push, anúncios e redes sociais, utilizando Data Cloud para unificar perfis e Einstein Content Selection para recomendações.

    Tópicos teóricos:

    • Marketing Cloud Connect e Data Cloud (Customer 360 Data Platform) para sincronização com Sales/Service Cloud e unificação de perfis.
    • Advertising Studio, Social Studio e MobilePush: criação de audiências, lookalikes e execução de campanhas em canais pagos e sociais.
    • Interaction Studio (Personalization) e Web Marketing: personalização em tempo real de sites e aplicativos, integração com dados comportamentais.
    • Einstein: Content Selection, Recommendations e AI generativa para criar conteúdo e determinar o próximo melhor passo.
    • Arquitetura e governança: design de soluções multicloud, gestão de volume de dados (milhões de contatos), privacidade, compliance e melhores práticas de deploy.

    Hands‑on / Labs:

    • Configurar o Marketing Cloud Connect para sincronizar Leads e Contacts da Sales Cloud.
    • Criar uma audiência no Advertising Studio e executar uma campanha de lookalike no Facebook.
    • Configurar regras de Einstein Content Selection para recomendações dinâmicas em emails e landing pages.
    • Implementar uma jornada multicanal com fallback: por exemplo, se um email não for aberto, enviar uma notificação push ou SMS.
    • Usar o Web Analytics Connector para capturar dados de navegação no site e personalizar interações subsequentes.

    Testes de conhecimento & Exercícios:

    • Quais considerações arquiteturais são necessárias para suportar campanhas para 5 milhões de contatos?
    • Descreva como o Data Cloud unifica perfis de clientes e como isso se integra ao Marketing Cloud.
    • Projete um fluxo de jornada que utilize email, push, SMS e anúncios pagos com base em comportamento de compra.
    • Documente a arquitetura de uma campanha omnicanal com fluxograma e explique cada etapa.
