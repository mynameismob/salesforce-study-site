    Sales Cloud

    Sales Cloud é a plataforma de automação de vendas que gerencia todo o ciclo de vendas, da geração de leads à conversão em clientes, contas e oportunidades. Nesta página você encontra o conteúdo completo do plano de estudo para cada nível de proficiência.

    Nível Iniciante

    Explicação: Sales Cloud fornece ferramentas para gerenciar leads, contas, contatos e oportunidades. O objetivo nesta fase é entender o ciclo de vida de um lead até se tornar uma oportunidade e configurar a nuvem para suportar o processo de vendas.

    Caso de uso: Uma equipe comercial quer rastrear potenciais clientes (leads), atribuir responsáveis e converter em oportunidades ao confirmar interesse.

    Tópicos teóricos:

    • Lead Lifecycle: criação de leads, regras de atribuição, Web‑to‑Lead.
    • Gerenciamento de Oportunidades: etapas (stages), probabilidade e datas de fechamento.
    • Modelo de Contas e Contatos: Person Accounts versus Business Accounts e hierarquias.
    • Record Types e Page Layouts: personalização da experiência por perfil.

    Hands‑on Lab:

    • Ativar Web‑to‑Lead e gerar um formulário HTML de exemplo. Publicar em um site de teste e validar criação de leads.
    • Criar regra de atribuição de leads para roteamento automático para vendedores.
    • Construir um Process Builder ou Flow que converta leads com o campo Interested__c = verdadeiro em oportunidades.
    • Customizar o Page Layout de Oportunidades com campos Stage, Amount e Close Date.

    Testes de Conhecimento: Explique a diferença entre Lead e Opportunity. Quando usar Record Type em vez de apenas Page Layout?

    Exercícios Práticos:

    • Criar relatório de leads por fonte e agendar envio diário para a gerência.
    • Configurar um Quick Action para criar uma oportunidade diretamente na página de Account.

    Nível Intermediário

    Explicação: As ferramentas de automação e previsão permitem escalar processos de vendas e obter insights em tempo real.

    Caso de uso: A meta mensal exige alertas automáticos quando oportunidades de alto valor são criadas.

    Tópicos teóricos:

    • Process Builder vs Flow: quando escolher cada um para atualizar campos e enviar notificações.
    • Processos de aprovação: fluxos de aprovação para descontos e condições especiais.
    • Forecasting: quotas, gerenciamento de territórios e ajustes manuais.
    • Joined e Matrix Reports: análise comparativa de pipeline versus vendas realizadas.

    Hands‑on Lab:

    • Criar um processo de aprovação para descontos superiores a 20% em oportunidades.
    • Modelar um Flow para postar mensagem no Chatter quando uma Opportunity maior que R$100k é criada.
    • Montar um Joined Report combinando dados de Opportunities e Orders.

    Testes de Conhecimento: Quando escolher Flow em vez de Process Builder? Como as quotas são atribuídas no Territory Management?

    Exercícios Práticos:

    • Configurar um Dashboard dinâmico exibindo pipeline por região.
    • Criar um Flow que atualize diariamente o campo DaysInStage.

    Nível Avançado

    Explicação: Desenvolvimento e integrações elevam o Sales Cloud a sistemas corporativos, permitindo lógica customizada e sincronização de dados externos.

    Caso de uso: Sincronizar oportunidades com um ERP via API para criar registros de faturamento automaticamente.

    Tópicos teóricos:

    • Apex Triggers: bulkification, variáveis de contexto e boas práticas.
    • Batch Apex, Queueable e Schedulable: processamento de grandes volumes em jobs agendados.
    • Named Credentials e Auth Providers: configurar OAuth para callouts REST.
    • CPQ Light: produtos, bundles e regras de preço (se a funcionalidade estiver disponível).

    Hands‑on Lab:

    • Escrever um Trigger after update em Opportunity que faça um HTTP callout REST ao ERP para criar uma ordem de faturamento.
    • Configurar Named Credential e Auth Provider para autenticação OAuth com o sistema externo.
    • Implementar um Batch Apex para revalidar oportunidades semanalmente.
    • Criar testes unitários com uso de HTTP mock para callouts.

    Testes de Conhecimento: O que é bulkification e por que é essencial em triggers? Quais os limites de callout e como mitigá-los?

    Exercícios Práticos:

    • Desenvolver um componente LWC que exiba dados de Opportunity usando @wire Apex.
    • Ajustar um Trigger para usar Queueable Apex e evitar limites de tempo.

    Nível Especialista/Consultor

    Explicação: Arquiteturas de vendas corporativas exigem design do modelo de dados, regras de compartilhamento e governança de deploy.

    Caso de uso: Implantação em uma grande empresa com divisões de vendas independentes, cada uma com regras de acesso e territórios próprios.

    Tópicos teóricos:

    • Territory Management 2.0: regras de atribuição de contas e territórios sobrepostos.
    • Modelo de Segurança: OWD, regras de compartilhamento e Apex sharing.
    • Release Management: Salesforce DX, pacotes desbloqueados e pipelines CI/CD.
    • Estrategias para grandes volumes de dados: skinny tables, queries seletivas e indexação.
    • Uso do Platform Encryption para campos sensíveis (ex.: Commission__c).

    Hands‑on Lab:

    • Modelar três divisões de vendas usando Territory Management, definindo regras de atribuição de contas.
    • Configurar um pipeline CI/CD em GitHub Actions com SFDX para deploy entre ambientes (Dev → QA → Prod).
    • Aplicar Platform Encryption a um campo sensível e validar acesso por perfis sem permissão.
    • Documentar as configurações de compartilhamento e a hierarquia de papéis em um diagrama.

    Testes de Conhecimento: Como o Territory Management difere da Role Hierarchy? Quais tipos de campo podem ser criptografados pela Platform Encryption?

    Exercícios Práticos:

    • Criar um pacote desbloqueado para metadados customizados e testar o deploy em uma org de QA.
    • Simular rollback de mudanças usando a CLI do Salesforce DX.
