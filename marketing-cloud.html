    Marketing Cloud

    Marketing Cloud Engagement é a plataforma de marketing digital omnicanal da Salesforce. Ela abrange email, SMS, push, anúncios e personalização em tempo real. Esta página traz o plano de estudos completo por nível.

    Nível Iniciante

    Explicação: Email Studio e Data Extensions formam o núcleo para envios de email segmentados e personalizados.

    Caso de uso: Enviar uma newsletter semanal segmentada para clientes ativos.

    Tópicos teóricos:

    • Data Extensions: tipos (Standard, Filtered, Random) e definição de relacionamentos Sendable.
    • Email Studio: uso do Content Builder, blocos de conteúdo e templates.
    • Send Classifications e Delivery Profiles para controle de remetentes e regras de envio.

    Hands‑on Lab:

    • Criar uma Data Extension Sendable chamada "Newsletter_Subscribers" com campos EmailAddress, FirstName e IsActive.
    • Importar um CSV de 100 registros para a Data Extension.
    • Construir um email template simples com personalização %%FirstName%%.
    • Configurar um Send Flow para enviar o email para a Data Extension.

    Testes de Conhecimento: Qual a diferença entre uma Filtered Data Extension e um Data Filter? Por que usar Data Extensions em vez de Lists?

    Exercícios Práticos:

    • Configurar um teste A/B de subject line para 10% da lista.
    • Criar conteúdo dinâmico com AMPscript básico (por exemplo, exibir mensagem diferente se IsActive = true).

    Nível Intermediário

    Explicação: Automation Studio e Journey Builder orquestram envios e dados em escala, permitindo jornadas multicanal.

    Caso de uso: Onboarding de novos leads em uma jornada cross‑channel com email e SMS.

    Tópicos teóricos:

    • Automation Studio: atividades de Import File, SQL Query e File Transfer.
    • Journey Builder: entry events, wait activities, decision splits e engagement splits.
    • AMPscript básico: uso de personalization strings e LookupRows para buscar dados.

    Hands‑on Lab:

    • Configurar uma automação de importação para carregar leads diários via FTP.
    • Criar uma Journey com entrada em Data Extension, email de boas‑vindas, espera de 2 dias e envio de SMS de lembrete.

    Testes de Conhecimento: Quando usar uma atividade SQL em vez de Data Filter? Qual a diferença entre Engagement Split e Decision Split?

    Exercícios Práticos:

    • Implementar AMPscript que exiba um código de oferta único para cada contato.
    • Configurar o Mobile Studio para envio de SMS dentro da Journey.

    Nível Avançado

    Explicação: Scripting e APIs permitem lógica customizada e integrações em tempo real para campanhas altamente personalizadas.

    Caso de uso: Capturar um evento de compra no e‑commerce e disparar um email transacional.

    Tópicos teóricos:

    • SSJS & WSProxy: manipulação programática de Data Extensions e envios.
    • REST API: utilização para Triggered Sends e recuperação de dados.
    • Deliverability: configuração de SPF/DKIM/DMARC, planos de warming de IP e monitoramento de engajamento.

    Hands‑on Lab:

    • Escrever um Script Activity em SSJS para limpar registros antigos em uma Data Extension.
    • Configurar uma Triggered Send Definition e executar uma chamada de API simulada.
    • Atualizar registros DNS (fictícios) com SPF e DKIM e validar usando ferramentas de verificação.

    Testes de Conhecimento: Como configurar autenticação OAuth para a REST API do Marketing Cloud? Quais métricas acompanhar para avaliar deliverability?

    Exercícios Práticos:

    • Escrever um script SSJS que envie um email de follow‑up para contatos que não abriram o email em 7 dias.
    • Documentar um plano de warming de IP para um novo IP dedicado.

    Nível Especialista/Consultor

    Explicação: Estratégias omnichannel e AI escalam personalização e engajamento em grande escala.

    Caso de uso: Lançar uma campanha multicanal (email, SMS, push, ads) para 5 milhões de contatos.

    Tópicos teóricos:

    • Orquestração Multicanal: uso de Ads Studio, MobilePush, SMS e Email em conjunto.
    • Einstein Recommendations & Content Selection para recomendações de produtos e conteúdo.
    • Integrações: Marketing Cloud Connect, Web Analytics Connector e Data Cloud para unificação de perfis.

    Hands‑on Lab:

    • Configurar o Marketing Cloud Connect para sincronizar contatos do Sales Cloud.
    • Implementar a sincronização de audiências no Ads Studio para criar um lookalike no Facebook.
    • Criar regras de Einstein Content Selection para recomendar produtos de acordo com o comportamento do usuário.
    • Desenvolver uma Journey multicanal com lógica de fallback (por exemplo, se o email não for aberto, enviar SMS).

    Testes de Conhecimento: Como o Web Analytics Connector impacta os links de email? Quais critérios definem uma audiência eficaz para lookalike?

    Exercícios Práticos:

    • Configurar uma automação que extraia relatórios e os envie para stakeholders semanalmente.
    • Documentar a arquitetura de uma campanha omnicanal com fluxograma e explicar cada etapa.
