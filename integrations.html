Integrações e Desenvolvimento Multi‑Cloud

A Salesforce é uma plataforma extensível que se conecta facilmente a sistemas externos. Esta seção resume os principais conceitos de integração e desenvolvimento que aparecem em várias nuvens do nosso plano de estudo.

APIs e Named Credentials

Para integrar o Salesforce com outros sistemas, você pode utilizar diferentes APIs (REST, SOAP, Bulk) e criar autenticações seguras via Named Credentials.

  • Use Apex para realizar callouts HTTP em classes e triggers. Sempre separe lógica de callout em classes reutilizáveis.
  • Configure Named Credentials para armazenar URLs e credenciais de autenticação, facilitando a manutenção e segurança.
  • Quando lidar com grandes volumes de dados, considere a API Bulk e os limites de consumo.

Padrões de Integração

Diferentes padrões servem a diferentes casos de uso. Escolha o padrão adequado considerando volume, latência e consistência.

  • Processo Batch: Exportar/importar grandes volumes de dados de forma periódica usando Data Loader ou integradores como MuleSoft.
  • Requisição‑Resposta: Callouts síncronos para serviços externos (por exemplo, obter status de implementação de um projeto).
  • Event Driven: Usar Platform Events e Change Data Capture para transmitir alterações em tempo real para outros sistemas.
  • Remote Process Invocation: Utilizar External Services e Flow Builder para orquestrar chamadas a APIs externas sem código.

Ferramentas de Integração

A Salesforce fornece e integra com diversas ferramentas para simplificar integrações complexas.

  • MuleSoft: Plataforma de integração que conecta Salesforce a aplicações on‑premise e SaaS via API‑led connectivity.
  • Salesforce Connect: Permite acessar dados externos em tempo real via objetos externos, reduzindo a necessidade de armazenamento.
  • Heroku: Use para construir microsserviços e aplicativos complementares que se conectam ao Salesforce via Heroku Connect.
  • Data Cloud: Anteriormente CDP, unifica dados de várias fontes para criar um perfil único de cliente, facilitando integrações multi‑canal.

Boas Práticas de Desenvolvimento

Independente da nuvem, algumas boas práticas garantem que integrações e customizações sejam escaláveis.

  • Escreva código Apex bulkificado e testado, evitando SOQL em loops e respeitando limites de governança.
  • Utilize frameworks de Trigger para centralizar lógica e evitar recursões.
  • Planeje e automatize deploys com Salesforce DX e pipelines CI/CD (por exemplo, GitHub Actions ou GitLab CI).
  • Mantenha documentação clara de endpoints, fluxos de dados e credenciais.
