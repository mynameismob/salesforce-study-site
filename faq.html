Perguntas Frequentes

Esta seção reúne perguntas comuns sobre o ecossistema Salesforce. Clique em uma pergunta para revelar a resposta.

Qual a diferença entre Profile e Permission Set?
Profiles determinam o nível de acesso padrão de um usuário (permissões de objetos, campos e configurações básicas) e são obrigatórios. Permission Sets são pacotes adicionais de permissões que podem ser atribuídos a vários usuários para expandir o acesso sem alterar o perfil base.
Quando devo usar Master‑Detail em vez de Lookup?
Use Master‑Detail quando o registro filho não puder existir sem o registro pai e quando você precisar de roll‑up summary fields no objeto pai. Lookup é apropriado para relacionamentos opcionais ou quando os registros devem ser independentes.
O que é o Flow Builder e como ele difere do Process Builder?
Flow Builder é a ferramenta declarativa mais poderosa da Salesforce para automatizar processos, permitindo criar fluxos complexos (tela, registro, autolaunched). Process Builder é uma ferramenta mais antiga focada em automações simples e está sendo gradualmente substituída pelos Flows.
Como funcionam os relatórios Joined e Matrix?
Relatórios Joined combinam dados de diferentes objetos ou conjuntos de relatórios em seções horizontais distintas, enquanto os Matrix permitem agrupar dados por linhas e colunas simultaneamente, facilitando comparações cruzadas de métricas.
Qual a principal função do Data Cloud (Customer Data Platform)?
O Data Cloud, anteriormente conhecido como Customer Data Platform, unifica dados de clientes de várias fontes para criar um perfil unificado. Isso permite orquestrar campanhas de marketing, vendas e serviço de forma consistente e personalizada.
Por que é importante a cobertura mínima de 75% em testes Apex?
A Salesforce exige que pelo menos 75% do código Apex em uma organização esteja coberto por testes para permitir deploys em produção. Isso incentiva práticas de teste e garante que o código seja confiável e mantenha a integridade da plataforma.
